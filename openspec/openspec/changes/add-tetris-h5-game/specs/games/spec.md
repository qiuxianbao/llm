## ADDED Requirements

### Requirement: 游戏初始化
游戏应在页面加载后正确初始化，显示空的游戏区域和初始分数。

#### Scenario: 页面加载完成
- **WHEN** 用户访问游戏页面
- **THEN** 显示一个空白的游戏区域和初始分数为0

### Requirement: 方块生成和下落
游戏应按一定时间间隔生成新的方块，并使其持续下落直到碰到障碍物。

#### Scenario: 生成新方块
- **WHEN** 游戏开始或当前方块固定后
- **THEN** 在游戏区域顶部中央生成一个新的随机形状的方块

#### Scenario: 方块自动下落
- **WHEN** 方块未碰到下方障碍物时
- **THEN** 方块应按设定的时间间隔向下移动一格

### Requirement: 方块控制
玩家应能够通过键盘控制方块的移动、旋转和加速下落。

#### Scenario: 左移操作
- **WHEN** 玩家按下左箭头键
- **THEN** 当前方块应向左移动一格（如果移动路径无障碍）

#### Scenario: 右移操作
- **WHEN** 玩家按下右箭头键
- **THEN** 当前方块应向右移动一格（如果移动路径无障碍）

#### Scenario: 旋转操作
- **WHEN** 玩家按下向上箭头键
- **THEN** 当前方块应按顺时针方向旋转90度（如果旋转后无重叠）

#### Scenario: 加速下落
- **WHEN** 玩家按下向下箭头键
- **THEN** 当前方块应立即向下移动一格（或快速下降）

### Requirement: 碰撞检测
游戏应准确检测方块与游戏边界或其他已固定方块的碰撞。

#### Scenario: 下方碰撞
- **WHEN** 方块下落时碰到其他方块或底部边界
- **THEN** 当前方块应停止移动并固定在当前位置

#### Scenario: 侧边碰撞
- **WHEN** 方块尝试向左或向右移动时碰到边界或其他方块
- **THEN** 移动操作应被阻止，方块保持原位

### Requirement: 行消除
当一行或多行被完全填满时，应将其消除并更新分数。

#### Scenario: 行消除
- **WHEN** 至少有一行被方块完全填满
- **THEN** 应消除这些行并将上方的所有方块向下移动相应行数

### Requirement: 计分系统
游戏应根据消除的行数计算分数，并实时更新显示。

#### Scenario: 分数更新
- **WHEN** 成功消除一行或多行
- **THEN** 应根据消除的行数增加相应分数并更新显示

### Requirement: 游戏结束判定
当新方块生成位置已有其他方块占据时，游戏应结束。

#### Scenario: 游戏结束
- **WHEN** 新方块生成时其初始位置已被占用
- **THEN** 游戏应停止，显示最终分数和结束信息
